# Запуск проекта на другом ПК

## 1. Что понадобится

- Docker Desktop (рекомендуемый способ запуска)
- интернет для первой сборки образа
- свободный порт `3000`

## 2. Перенос проекта

Перенеси проект на другой ПК одним из способов:

1. Через Git:
   - `git clone <URL_репозитория>`
2. Через архив:
   - скопируй всю папку проекта целиком

После этого открой терминал в корне проекта (где лежит `docker-compose.yml`).

## 3. Быстрый запуск одной командой (рекомендуется)

```bash
docker compose up --build
```

После запуска сайт будет доступен по адресу:

- [http://localhost:3000](http://localhost:3000)

Что происходит при первом запуске:

- поднимается backend (Express)
- собирается frontend (React)
- создается/инициализируется SQLite база
- загружаются варианты и задания из `server/src/seed-data.json`

## 4. Запуск в фоне

Если нужно запустить в фоне:

```bash
docker compose up -d --build
```

Проверить логи:

```bash
docker compose logs -f
```

Остановить:

```bash
docker compose down
```

## 5. Полный сброс данных (если нужно начать с нуля)

```bash
docker compose down -v
```

Затем снова:

```bash
docker compose up --build
```

## 6. Проверка, что всё работает

Открой в браузере:

- [http://localhost:3000](http://localhost:3000)

Проверь:

- режим тренажера (задания 1–15)
- полный вариант с таймером `2:30`
- задания второй части (13–15) отображаются и помечены как ручная проверка
- картинки в заданиях отображаются

## 7. Частые проблемы

### Порт 3000 занят

Освободи порт `3000` или измени проброс порта в `docker-compose.yml`:

```yaml
ports:
  - "3001:3000"
```

Тогда сайт будет на [http://localhost:3001](http://localhost:3001).

### Docker Desktop не запущен

Запусти Docker Desktop и повтори команду `docker compose up --build`.

### Ошибки сети при сборке

Повтори команду сборки:

```bash
docker compose up --build
```

Обычно это временная проблема подключения к Docker Hub.

## 8. Запуск без Docker (опционально)

Требуется Node.js 20+.

### Backend

```bash
cd server
npm install
npm run dev
```

### Frontend

В другом терминале:

```bash
cd client
npm install
npm run dev
```

Frontend будет на [http://localhost:5173](http://localhost:5173), API проксируется на backend.
